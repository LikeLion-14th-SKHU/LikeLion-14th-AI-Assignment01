# LikeLion-14th-Assignment-Template
🦁 SKHU 멋쟁이사자처럼 14기 과제 PR 템플릿 레포지토리입니다.

1. 선택한 서비스- 멜론(Melon) 음악추천
2. 이 서비스는 무엇을 예측하는지
-멜론 사용자가 특정 노래를 끝까지 들을 확률이나 그 노래에 좋아요 버튼을 누를 가능성을 예측합니다., 현재 사용자의 상황(시간, 날씨 등)에서 사용자가 가장 만족할 만한 다음곡을 예측하여 추천합니다.
3. 어떤 data가 사용될까(최소 3가지 이상 작성)
-청취 패턴 데이터: 어느 특정 곡을 반복재생해서 듣는다거나 도입부에서 더이상 추가로 듣지않고 건너뛰기를 하는 기록입니다.
곡의 데이터(특징): 곡의 템포, 장르, 가수, 가수의 목소리 톤, 사용된 악기구성 등등 모델이 배우는 데이터가 됩니다.
상황 정보 데이터: 평소 사용자가 음악을 듣는 시간대(예시: 출근길, 새벽, 카페공부 등), 현재 위치나 날씨나 계절 등의 외부 환경 데이터입니다.
4. 모델은 어떻게 학습할까?
-예측과 오차를 확인: 모델이 추천한 곡을 사용자가 중간에 넘기거나 끄면 예측이 틀렸다고 판단하여서 Loss(오차)를 계산합니다.
숫자 수정하기: 음수를 제거하고 큰 패널티를 주기 위해서 오차를 제곱하여서 계산하며 이 Loss를 줄이는 방향으로 모델의 내부의 가중치와 편향을 수정합니다.
경사하강법 적용하기: 학습률에 따라 조금씩 숫자를 바꿔가면서 최적의 추천값을 찾아가는 과정을 계속 반복합니다.
5. 이 모델이 더 좋아지려면 어떻게 해야할까?
-딥러닝 활용하기: 단순하게 장르만 분류하는것이 아닌 곡의 자세한 분위기와 사용자의 취향을 깊고 자세하게 패턴을 파악할 수 있도록 계산 층을 여러 층으로 깊게 쌓는 신경망을 활용합니다.
데이터 다양성 확보하기: 사용자가 평소에 즐겨듣지않던 새로운 장르의 데이터도 학습시켜서 모델이 익숙한 노래만 계속 추천하는 과적합 상태에 빠지지 않도록 관리합니다.
하이퍼파라미터 최적화하기: 학습 횟수나 배치크기같은 하이퍼파라미터를 정교하게 설정해서 모델의 성능을 극대화합니다.`
